# Решение проблем

## Ошибка: TelegramUnauthorizedError: Unauthorized

Эта ошибка означает, что бот не может авторизоваться в Telegram API.

### Причины и решения:

#### 1. Файл .env отсутствует или не найден

**Решение:**
- Создайте файл `.env` в корне проекта (рядом с `main.py`)
- Добавьте в него:
```env
BOT_TOKEN=ваш_токен_бота
ADMIN_ID=586797053
ADMIN_IDS=586797053
AI_API_KEY=ваш_ai_api_key
```

#### 2. BOT_TOKEN не установлен или неверный

**Решение:**
1. Получите токен у [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/newbot` или `/token` для существующего бота
3. Скопируйте токен и вставьте в `.env`:
```env
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
```

#### 3. Токен неверный или отозван

**Решение:**
- Проверьте, что токен скопирован полностью (без пробелов в начале/конце)
- Убедитесь, что токен соответствует правильному боту
- Если токен был отозван, получите новый у @BotFather

#### 4. Файл .env не загружается

**Решение:**
- Убедитесь, что файл называется именно `.env` (с точкой в начале)
- Проверьте, что файл находится в корне проекта
- Убедитесь, что установлен пакет `python-dotenv`: `pip install python-dotenv`

### Проверка настройки:

1. Проверьте наличие файла:
```bash
ls -la .env  # Linux/Mac
dir .env     # Windows
```

2. Проверьте содержимое (НЕ коммитьте реальные токены в Git!):
```bash
cat .env     # Linux/Mac
type .env    # Windows
```

3. Убедитесь, что токен правильный:
- Токен должен начинаться с цифр и содержать двоеточие
- Формат: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`

### Пример правильного .env файла:

```env
# Bot Token (получите у @BotFather)
BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz

# Admin IDs (через запятую)
ADMIN_ID=586797053
ADMIN_IDS=586797053

# AI API Key
AI_API_KEY=your_ai_api_key_here

# Database Configuration
DATABASE_URL=
```

### На сервере (Linux):

Если вы запускаете на сервере, убедитесь:
1. Файл `.env` скопирован на сервер
2. Права доступа правильные: `chmod 600 .env`
3. Файл находится в правильной директории (там, где запускается `main.py`)
